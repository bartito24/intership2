Informacion General INTERNSHIP

Este es el Manual de referencia para el sistema de INTERNSHIP. Para obtener información sobre el sistema y sus veneficios como tambien desventajas, consulte los Descripcion.

Este manual no está diseñado para usarse con otros sistemas que no sea sobre seguimiento y control de pasantia. Si está utilizando otro sistema, verifique que sea

Descripcion

Este software implementado para la realización y facilidad en el momento de hacer un control mas exaustivo en la toma de pasantia del estudiante y la entrega de pasantia culminada. El sistema digitalmente controlara toda función de seguimiento de pasantia para no perder informacion de todos los estudiantes de la institucion INFOCAL.

La información contenida en este documento está sujeta a cambios sin previo aviso y no se garantiza que esté libre de errores. Si encuentra algún error, infórmenos por escrito.

Si se trata de un software o documentación relacionada que se entrega a INFOCAL o cualquier otra persona que lo otorgue en su nombre, entonces se aplica el siguiente aviso:

USUARIOS FINALES DE INFOCAL: Los programas de INTERNSHIP, incluidos cualquier sistema operativo, software integrado, cualquier programa instalado en el hardware y / o la documentación, entregado a INFOCAL, Son "software informático" de conformidad con el Reglamento de Adquisiciones aplicables. Reglamentos complementarios específicos. Como tal, el uso, la duplicación, la divulgación, la modificación y la adaptación de los programas, incluido cualquier sistema operativo, software integrado, cualquier programa instalado en el hardware y / o documentación, estarán sujetos a los términos de las normas y las restricciones de los programas.

Este software o hardware está desarrollado para uso general en una variedad de aplicaciones de administración de información. No está desarrollado ni diseñado para usarse en aplicaciones intrínsecamente peligrosas, incluidas las aplicaciones que pueden crear un riesgo de lesiones personales. Si usa este software o hardware en aplicaciones peligrosas, entonces será responsable de tomar todas las medidas de seguridad, redundancia y otras medidas adecuadas para garantizar su uso seguro.

El sistema tendra varias funciones deacuerdo al cargo o rubro designado, mostrara requisitos de adquisición para la carta de pasantia, mostrara requisitsitos para la entrega de la pasantia culminada, generara visitas, registrara documentos de culminacion de la pasantia del estudiante, controlara las visitas del tutor o docente acargo, generara un evaluador de desempeño del estudiante en la empresa que realizara su pasantia, tambien generara reportes de las distintas funcionalidades que realizaran los usuarios.
